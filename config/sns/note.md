# note 投稿ルール

## プラットフォームの性格

**「信頼」と「検索流入」のプラットフォーム**。
Google / note 内検索から読者が来る。読み物 / ブログとして成立させる必要あり。

## 重要: アプリの役割 (案 3 採用、2026-04-22 確定)

note は**完全に手動投稿**。アプリはブラウザ自動化を一切行わない。

アプリができること:
1. 「投稿」ボタン押下時、本文を**クリップボードに自動コピー**
2. 「note で開く」ボタン (投稿ボタン隣) 押下で、OS 既定ブラウザで
   `https://note.com/notes/new` を開く

妻のやること:
1. note タブで本文エリアにカーソルを置いて **Ctrl+V** (= 貼り付け)
2. タイトルを note 上部欄に入力 (アプリで生成済のタイトルもクリップボード履歴から取れる設計)
3. 画像を手動でドラッグ&ドロップ or アップロード (note 側の機能)
4. note 画面の「公開」ボタンを押す

### なぜ案 3 か
note 規約 13 条の自動化制限を完全に回避するため。アプリは HTTP 要求を
note に送信しないし、note のページ DOM も触らない。OS レベルの
「URL を既定ブラウザで開く」操作のみ。規約違反の余地ゼロ。

詳細は ADR-001 Revisions 2 巡目 §4、ADR-006 参照。

## 仕様制約

- max_chars: 実質無制限 (1500-3000 字がスイートスポット)
- hashtag: 記事末尾に 3-5 個
- 画像: 見出し画像 1 枚 + 本文内 2-3 枚推奨
  - v0.1 では妻が note 画面で手動アップロード (自動化しない)
  - アプリ側では画像選択状態をプレビューのみ表示
- リンク: 本文中に自然に複数埋め込み OK
- **タイトル: 32 文字以内** (SEO 最適値、検索結果で切れない)

## 文体

- 丁寧体 (「〜です」「〜ます」)
- **1 人称は「私 / 妻」(aya さん本人の視点)**
- ブログ記事としての読み物感を重視
- 個人の体験 / 失敗談を入れると強い

## タイトル生成ルール (SEO)

フォーマット: **`商品名 + メリット + 感想`** を 32 文字以内に収める

例:
- ✅ `◯◯ホットクック、ズボラ主婦が1ヶ月使った本音レビュー`  (30字)
- ✅ `3児ママが選ぶ◯◯電動歯ブラシ、子供が自分で磨くように` (28字)
- ❌ `すごい商品を紹介します` (SEO 価値ゼロ、商品名なし)

## 構成テンプレ (H2/H3 見出しで Google に好かれる構造に)

```markdown
# [商品名] + [メリット] + [感想] (32字以内)

## 導入 (悩み共感)

[ここに妻の悩みの具体描写を入れる。3-4 行]

## 結論: こういう人に向いてる

- [箇条書き 3 つ]

## なぜこの商品を選んだか

[選定理由、具体的な比較 2-3 段落]

### こんな期待をしてた

[購入前の期待を書く]

## 使ってみた Before / After

### Before (買う前の状況)

[[ここを入力: 購入前の具体的な困り事]]

### After (買ってから変わったこと)

[[ここを入力: 実際に使って変わったこと、具体例]]

### 驚いた点 (生の声)

[[ここを入力: 使って初めて気付いたこと]]

## 買うときの注意点

- [注意点 2-3 個]

## まとめ

[締め + リンク]

---
#PR #ジャンルタグ1 #ジャンルタグ2
```

## 「空欄 [ここを入力]」のルール (実体験捏造防止)

LLM に**実体験部分を捏造させない**。代わりに `[[ここを入力: ...]]` 形式で
**雛形を生成**し、妻が自分の言葉で埋める設計:

- 「実際に使ってみて」の具体描写
- 「ここが驚いた」系の感想
- Before / After の具体事例

LLM は「枠組み」だけ作り、**固有の体験は空欄**にする。
この方針を system prompt に明記する (ADR-009 SNS 別コンテンツ生成エンジンで扱う)。

## ハッシュタグ戦略

- 検索流入狙い: ジャンルタグ 3-4 個 (note 内検索対応)
- #PR (必須)
- 例: `#子育て` `#時短家事` `#買ってよかったもの` `#PR`

## NG

- note は検索エンジンからも読まれる → 薬機法 / 景表法の表現により厳格に
- タイトルで釣って中身薄い記事 (note の SEO ペナルティ対象)
- 実体験を LLM に捏造させる (発覚すると信頼失墜)
- 「絶対」「必ず」などの断定表現

## LLM プロンプトに含める指針

- タイトルは 32 字以内、「商品名 + メリット + 感想」の順
- H2/H3 見出しを必ず使う
- 実体験部分は `[[ここを入力: 具体的な描写指示]]` で空欄化
- Before / After の枠組みを必ず用意
- 1500-3000 字を目安
