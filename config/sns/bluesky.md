# Bluesky 投稿ルール

## プラットフォームの性格

**「専門性」と「カスタムフィード」のプラットフォーム**。
テック / ニッチ層が多く、煽り系は嫌われる。正確なタグでフィードに拾われる設計。

## 仕様制約

- max_chars: 300 (grapheme count、日本語でおよそ 300 文字)
- **hashtag_limit: 2-4 個** (カテゴリ名を**正確に**)
- 画像: 最大 4 枚、1 枚 1MB まで (超過は Pillow で自動圧縮)、jpg/png (v0.1 対応)
- **画像には Alt テキスト必須** (アクセシビリティ、v0.1 で LLM に生成させる)
- リンク: **1 つに絞る** (OGP カードが綺麗に出る、複数貼ると全部表示崩れ)

## 文体

- 丁寧、または少しマニアックな解説調
- 1 人称は「私」
- 絵文字は少なめ (1-3 個)
- 数値 / 具体スペックを 1 つは盛り込む (例: 「約 200g で片手で持てる」)

## 構成テンプレ

```
[共感 or 小さな発見 1-2 行]
[具体スペック or 技術的な 1 事実 1 行]
[使用感 / 解決したこと 2-3 行]
[URL を 1 行だけ裸貼り]
#カテゴリタグ1 #カテゴリタグ2 #PR
```

## ハッシュタグ戦略

- **正確なカテゴリ名**を 2-4 個 (カスタムフィード拾いの要)
- 例: `#ガジェット` `#キッチン便利グッズ` `#子育て` `#時短`
- 独自の造語タグは避ける (フィードに乗らない)
- #PR (必須)

## Alt テキスト (画像添付時、v0.1 対応)

- 画像アップロード時、LLM で Alt テキストを自動生成
- 視覚情報を文字で説明 (読み上げ対応)
- 例: 「白いシンプルなデザインの電気ケトル、注ぎ口が細く、木目調のテーブルに置かれている」

## NG

- 他 SNS のテキストをそのままクロスポスト (Threads/X のノリは嫌われる)
- 過剰な絵文字
- 誇大表現 / 煽り
- 自作タグで固める (フィードに乗らない → 届かない)

## LLM プロンプトに含める指針

- 具体スペックを最低 1 つ含める
- 文字数 250-300 に収める
- 絵文字は 0-3 個
- リンクは 1 つだけ
- カスタムフィード拾い狙いの正確なカテゴリタグを 2-4 個
- 画像があれば Alt テキストも別フィールドで生成
