# Bluesky 投稿ルール (2026 年アルゴリズム対応版)

## プラットフォームの性格

**「情報の透明性」と「カスタムフィードの力」**。
時系列順の直接フィード + ユーザーが好むキーワードを購読する「カスタムフィード」文化。
エンジニア / クリエイター / 情報感度高い層が多く、アクセシビリティに敏感。

**Threads と真逆の特徴 (重要):**

| 項目 | Bluesky | Threads |
|---|---|---|
| リンク貼り | **ペナルティ ほぼなし** (OGP カードが自動で綺麗に) | 親ポストに入れると 80% カット |
| ハッシュタグ | **必須** (0-4 個、カスタムフィード拾いの要) | 0-1 個 (多いとスパム判定) |
| おすすめ/フィード | 時系列 + カスタムフィード (ユーザー主導) | AI おすすめ (機械主導) |
| 滞在時間 | 重要度 中 | S 級シグナル |

## 仕様制約

- max_chars: **300** (grapheme count、日本語でおおよそ 300 文字)
- **hashtag_limit: 2-5 個** (ジャンル特化タグ中心、カスタムフィード拾いの要)
- 画像: 最大 4 枚、1 枚 1MB まで、jpg/png (v0.1 対応)
- **画像には必ず Alt テキスト** (アクセシビリティ文化、サボると bot 扱い → エンゲージ急落)
- リンク: **親ポスト本文に直接貼って OK** (OGP カードが綺麗に出る)。長文スレッドなら 1 つだけに絞るのが理想

## 絶対守るべき 6 則

1. **カスタムフィード狙いのジャンルタグを必ず入れる** (例: `#キャンプギア` `#買ってよかった` `#時短家事`)
2. **画像は必ず Alt テキスト付き** (LLM にも生成させる)
3. **リンクは本文に直貼り OK** (Threads と違いペナルティなし)
4. **「宣伝文丸出し」禁止** (エンジニア層に見破られる、具体スペックで語る)
5. **#PR / #広告 必須** (アフィ投稿時、景表法 + ユーザー信頼)
6. **投稿は時系列順表示される** → ゴールデンタイム (JST 朝 7-9、夜 21-23) に投稿

## 文体

- 丁寧、または少しマニアックな解説調
- 1 人称は「私」
- 絵文字は少なめ (1-3 個、0 でも OK)
- **具体スペック / 数値を必ず 1 つ含める** (「約 200g」「1 日 3 杯分沸く」等)
- 煽り系・情緒過多はむしろ逆効果 (落ち着き重視)

## 構成テンプレ (単発投稿、300 字)

```
[冒頭 1-2 行: シーン + キーワード (タグ拾い狙い)]
[具体スペック or 技術的事実 1 行]
[使用感 / 解決したこと 2-3 行]
[URL 1 行 (OGP カード自動表示)]
[#ジャンルタグ ×2-4] #PR
```

## 構成テンプレ (スレッド形式、深掘り・推奨)

売れるスレッドは 3 投稿構成:

- **1 投目**: 結論 + 写真 + ジャンルタグ (カスタムフィード拾い)
- **2 投目**: なぜ良いのかの理由 (主婦視点 / エンジニア視点 / 具体スペック)
- **3 投目**: アフィリンク + #PR

→ 滞在時間を稼ぎつつ、引用リポスト (Quote) を誘発するパターン。

## ハッシュタグ戦略 (Threads と違い複数 OK)

- **ジャンル特化タグ** 2-3 個 (カスタムフィード拾い、例: `#子育て` `#時短家事` `#キャンプ`)
- **#PR** 必須 (アフィ投稿のみ)
- **独自造語タグ** は避ける (誰もフォローしてないので届かない)

## Alt テキスト生成ルール (画像添付時、v0.1 対応)

- 画像アップロード時、LLM で Alt テキストを**自動生成**
- 視覚情報を文字で記述 (読み上げソフト対応)
- 例: 「白いシンプルなデザインの電気ケトル、注ぎ口が細く、木目調のテーブルに置かれている」
- **長すぎる alt は逆効果** (200 字以内目安)

## 分析 (v0.2 以降の実装予定)

Bluesky は公式分析ダッシュボードが弱い。代替指標をアプリ側で記録:

| 指標 | 重要度 | 記録方法 |
|---|---|---|
| 引用リポスト (Quote) 数 | ◎ 最強シグナル (「愛用してる」系は購買意図高) | 投稿 URI を保存、定期 polling で反応追跡 (v0.2) |
| リンククリック数 | ◎ 楽天アフィレポートと投稿時刻照合 | 投稿 URL にユニーク UTM 付与 (v0.2) |
| いいね数 | ○ | 参考値、Bluesky では Quote が優勢 |
| リポスト数 | △ | Bluesky は独自の会話より再共有も評価される |

## NG

- ハッシュタグなし (カスタムフィードに載らない = 届かない)
- 画像の Alt 未設定 (bot 扱い → エンゲージ落ち)
- Threads のノリそのままクロスポスト (フランクすぎ / 煽りすぎは嫌われる)
- 「リンクはプロフから」誘導 (Bluesky は本文リンク OK なので不要)
- 絵文字てんこ盛り (5 個以上はノイズ)

## LLM プロンプトに含める指針

- 具体スペック / 数値を最低 1 つ含める
- 文字数 250-300 に収める (300 超過は自動で切り詰め)
- ハッシュタグはジャンル特化 2-4 個、アフィ時は末尾に `#PR`
- 画像添付時は Alt テキストも別フィールドで生成 (各画像 200 字以内)
- **本文にアフィリンク 1 つ直接貼って OK** (Threads と違う)
- 煽り調 / 情緒過多禁止、落ち着いた語り口
- 毎回異なる導入で書く (過去投稿と類似度 70% 超禁止)
