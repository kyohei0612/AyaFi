# ADR-012: Threads 運用アルゴリズム戦略 (2026 年最新)

- **日付**: 2026-04-22
- **ステータス**: 提案中 (承認待ち)
- **決定者**: kyohei
- **関連**: [ADR-009](009-sns-content-generation-engine.md), [config/sns/threads.md](../../config/sns/threads.md)

---

## Context (問題)

Threads は「おすすめ (For You) フィード」に載るか否かで露出が**桁違い**に変わる。
外部リンク (アフィリンク) への厳しいペナルティ、Bot 検知、「人間らしさ」判定など、
単に LLM で文章を書くだけでは**アルゴリズムに殺される**設計になっている。

アフィ目的で投稿しても、以下が揃わないとフォロワー外に届かず成果にならない:
- **親ポストにリンクなし** (リンク入りは初期 15 分で露出 80% カット)
- **会話誘発** (返信 = S 級シグナル)
- **多様性** (コピペ文 / 同じ挨拶は Bot 判定)
- **人格の蓄積** (普段喋らない垢がいきなり広告すると拒絶される)

プロの観察に基づく 2026 年最新知見をアプリ側の挙動に組み込む。

## Decision (結論)

### 1. Threads 内部スコアリング (優先順位)

| ランク | アクション | 設計への影響 |
|---|---|---|
| **S** | 返信 (リプライ) | 会話誘発必須、投稿直後 15 分の返信対応が最重要 |
| **A** | 滞在時間 | 長文 or 画像必須、冒頭 2 行で惹きつける |
| **B** | 再投稿 | 独自の会話重視、他人の丸パクリは無視される |
| **C** | いいね | 依存しない、挨拶程度の指標 |

### 2. リンク取扱い (致命的)

- **親ポストにアフィリンク入り → 初期 15 分で AI が「宣伝」判定 → フォロワー外 露出 80% カット**
- 対策: **2 段階投稿** (親ポスト = 本文のみ / 自己リプ = アフィリンク)
  - 実装: `poster/threads.py` で `create_container(main) → publish → create_reply(url, parent_id) → publish`
  - v0.1 で Stage 3 の Threads 投稿実装時に必須

### 3. 人間らしさ判定 (Bot 回避)

| 判定項目 | アプリ側の対策 |
|---|---|
| 投稿速度 1 分以内複数投稿 | **投稿間隔 2 分以上の強制待機** (クライアント側 + API 側 rate limit) |
| コピペ率 (同じ定型文) | LLM プロンプトに「過去投稿との類似度 70% 以上禁止」を明記、履歴と diff チェック (v0.2) |
| リプライ無視 | 投稿から 15 分間は UI に「リプ通知を待つ」ダイアログ + リマインダー |

### 4. タグ / キーワード

- **タグは 1 投稿に 1 つだけ** (2 つ以上は逆効果)
- **キーワードは冒頭 2 行に埋め込む** (検索 + Google 流入補助)
- Validator に反映: `len(tags) <= 1` (警告でなく **error**)

### 5. 週次ルーティン (「準備期間」戦略)

| 曜日 | モード | 内容 | アフィリンク |
|---|---|---|---|
| **月〜木** | `preparation` (準備) | **魂の 5 大お題** (後悔 / 驚き / 日常の工夫 / 失敗談 / 季節もの) | **禁止** |
| **金〜日** | `affiliate` (本投稿) | 買い物欲ピークの週末に 2 段階投稿で投下 | 許可 (リプ側のみ) |

準備期間の目的:
- 普段からよく喋る人間だとアルゴリズムに覚えさせる
- ayaさんの顔と人格をフォロワー外に売る
- 週末の本投稿で「いきなり宣伝」扱いされないための**信用貯金**

### 6. 魂の 5 大お題 (準備期間の投稿テンプレ)

| # | お題 | 切り口例 |
|---|---|---|
| 1 | **後悔** | 最近買って後悔したもの / やらなくてよかったこと |
| 2 | **驚き** | 意外と知らなかった発見 / 常識が覆った瞬間 |
| 3 | **日常の小さな工夫** | 家事 / 子育ての時短テク、続けているルーティン |
| 4 | **失敗談** | 笑える / 共感できる失敗、キッチンでの大惨事 |
| 5 | **季節もの** | 今の時期に感じること / やりたいこと / 準備してること |

ルール:
- 同じお題でも**毎回切り口を変える**
- 必ず**質問**で締める (会話誘発)
- タグは **1 つだけ** (ジャンル系タグ、例: `#時短家事`)
- **画像推奨** (生活感のあるもの > 商品写真)

### 7. 成功確率式 (プロメンタルモデル)

```
成功確率 = (ayaさんの生の声 × 質問文) + (リプライ欄へのリンク配置 × ゴールデンタイム投稿)
```

- **生の声**: LLM が捏造せず、妻が空欄 (`[[ここを入力]]`) を埋めた実体験
- **質問文**: 本文末尾の問いかけ
- **リプライ欄リンク**: 2 段階投稿
- **ゴールデンタイム**: JST 7-9時 / 12-13時 / 20-22時

### 8. 15 分ルール

投稿直後の 15 分間:
- **誰かと会話できる** → 「おすすめ」に乗る確率大幅上昇
- 反応ゼロで 15 分経過 → ほぼ冷める

アプリ側実装 (v0.2):
- 投稿成功後、15 分のカウントダウン UI
- リプ通知を受けたら即返信できる「クイック返信」画面
- 15 分超過 → 「冷め時間」に突入と明示

v0.1 では:
- 投稿成功トーストに「15 分以内に 1 回返信しよう」と明記する

## Alternatives 検討

### 案 A (採用): 週次モード + 2 段階投稿 + Validator 強化
- ✅ アルゴリズム全 8 項目に対応
- ✅ 準備期間があるため「いきなり宣伝」失敗を回避できる
- ⚠ 妻の運用コスト: 月-木も投稿する必要がある
- ⚠ 月-木の投稿は成果直結しないので、妻のモチベ維持設計が別途必要

### 案 B: 毎日アフィ投稿 (現状の素朴実装)
- ❌ アルゴリズムに殺されて露出ゼロ
- ❌ 1 回の成功経験が積めないまま妻が諦める
- → 却下

### 案 C: 金-日のみ投稿 (週 3 日運用)
- ⚠ 投稿頻度低すぎてフィードから消える
- ⚠ 準備期間の「人格蓄積」が発生しない
- → 却下

### 案 D: 3 段階投稿 (親 + リプ + さらにリプ)
- ✅ リンクとコンテキストを分離できる
- ❌ API コール数 3 倍、レート制限に抵触しやすい
- ❌ 妻の運用が複雑化
- → v0.2 で再検討

## Failure Modes + 対処

| 失敗モード | 影響 | 対処 |
|---|---|---|
| 親ポストにアフィ URL 混入 | 露出 80% カット、1 週間分の努力が無駄 | Validator を `error` 判定 (投稿前にブロック)、UI で「URL は自動的にリプライ側に配置されます」と明示 |
| 妻が 15 分以内に返信しない | 「おすすめ」に乗らない | 投稿直後の UI カウントダウン + トースト、妻 PC の通知機能に接続 (v0.2) |
| LLM が毎回同じ導入文 | Bot 判定でスコア低下 | システムプロンプトで「過去 N 投稿と類似度 70% 超を禁止」+ 履歴 SQLite との diff チェック (v0.2、Stage 5 と連動) |
| タグ 2 つ以上 | スパム判定 | post-processor で 2 つ目以降を自動削除 + 警告表示 |
| 準備期間の投稿を妻がサボる | 信用貯金が溜まらず週末本投稿が沈む | UI で「今週の投稿数」を可視化、月-木で 2-3 投稿を目標提示 |
| 投稿間隔 1 分以内で Bot 判定 | フィード露出降格 | アプリ内でクライアント側強制インターバル (最後の投稿から 2 分未満なら投稿ボタン disable) |

---

## ロールバック手順

1. **戦略が合わない**: config/sns/threads.md を単純ルール版に戻す、mode 機能を UI から隠す
2. **2 段階投稿が煩雑**: 1 段階に戻し、URL は本文末尾に置く (露出犠牲を受け入れる)
3. **Bot 判定回避が効果ない**: 投稿間隔や重複チェックを緩和

---

## 計測 (成功判定)

### v0.1 リリース後 1 ヶ月
- **週次モード利用率**: 月-木の「準備」モード投稿が週 2 件以上
- **2 段階投稿率**: アフィ投稿が 100% リプ側にリンク配置
- **15 分ルール遵守率**: 投稿後 15 分以内の自己返信 or リプ返信率 50% 以上

### v0.2 以降 (履歴 + 分析実装後)
- **フォロワー外インプレッション比率** 向上 (準備モード効果の指標)
- **リプライ / いいね比** 向上 (S 級シグナル重視の効果)
- **アフィクリック数 / 投稿インプ** (CTR) 向上

## 影響範囲

- **ADR-009 (SNS 別生成エンジン)**: Threads の prompt_builder に mode 引数追加、
  Threads Validator を強化。
- **ADR-005 (投稿履歴)**: `posts` テーブルに `mode` / `post_type` カラム追加検討。
- **Stage 3 (SNS 実投稿)**: Threads Poster を 2 段階投稿対応に設計する必要あり。
- **v0.1 での部分実装**:
  1. ✅ UI にモード選択 (準備 / 本投稿)
  2. ✅ システムプロンプトをモード別に切替
  3. ✅ 準備モード = 魂の 5 大お題を選択 UI 化
  4. ⏸ 2 段階投稿 (Stage 3 で実装)
  5. ⏸ 15 分ルール UI (v0.2)
  6. ⏸ 過去投稿との diff チェック (Stage 5 + v0.2)
